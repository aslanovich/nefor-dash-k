# nefor-dash-k

Pixel-perfect прототип дашбордов **Альфа-Бизнеса**, мигрированный со статики на **Next.js 15**.
Цель проекта — точная вёрстка и «живые» анимации 1:1 с исходными макетами Figma, плюс
интерактивный AI-модуль (нейропомощник с 3D-кнопкой).

Форк `FatherofNations/nefor-dash`. Прод — **Vercel**, автодеплой из `main`.

---

## Стек

- **Next.js 15** (App Router) · **React 19** · **TypeScript 5**
- **three.js** (`^0.185`) — 3D-клавиша AI, ручной парсинг GLB без GLTFLoader
- Без UI-библиотек. Глобальный CSS перенесён вербатим из исходной статики
- Шрифты — локальные woff2 (`Alfa Interface Sans`, `Styrene UI`)
- Web Audio API — эквалайзер голосового режима; `<audio>`-пул — звук клика клавиши

## Запуск

```bash
npm install
npm run dev      # dev-сервер (:3000; в превью использовался -p 4177)
npm run build    # прод-сборка + проверка типов. НЕ запускать при живом dev — см. «Подводные камни»
npm start        # прод-сервер после build
```

## Роуты

| URL         | Что это |
|-------------|---------|
| `/`         | Главная («ИП»). Внутри — переключаемые варианты **v1** и **v2** (тоггл в панели tools, НЕ отдельный роут) |
| `/current`  | Дашборд «Бухгалтер» (новый макет) |

Состояние панели дип-линкуется в URL без перезагрузки: `/?menu=v2&stack=over,credit` —
`variant` и 6 переключателей стека читаются из `?menu`/`?stack` при заходе по ссылке и
пишутся через `history.replaceState` (без навигации/ре-фетча).

## AI-модуль (нейропомощник)

Смонтирован на всех дашбордах: `MainV1`, `MainV2` (`left=324 right=88 bottom=40`) и
`/current` (`left=300 right=52 bottom=40`). Три файла: разметка (`NeuroBar.tsx`),
поведение строки (`useNeuroBar.ts`), 3D-клавиша (`AiKeyButton.tsx` + `keyCapMesh.ts`).

### 3D-клавиша (`AiKeyButton.tsx`)

- **three.js-сцена** на прозрачном канвасе: ортокамера, покрашенный по вершинам keycap
  (розово-малиновый верх, красно-золотая боковина), надпись «АИ✦» текстурой, sparkle-спрайт.
- **Физика нажатия** — пружина (проседание на ~половину высоты), наклон к курсору через
  raycast, спарк-частицы при отпускании.
- **Клик** засчитывается по всему канвасу (а не только по мешу) → ведёт в `nbActivate()`
  (разворот строки) через `onActivate`/`activateRef`.
- **Перетаскивание** (`--dx`/`--dy` на `.aik-stage`): если между press/release сдвиг > 3px —
  это drag (двигает клавишу, НЕ активирует строку), иначе клик.
- **Хвост** при перетаскивании — непрерывная градиентная полоса из капсул-сегментов между
  точками пути, тает за ~2.4с. Тумблер «Хвост при перетаскивании» (по умолчанию **выкл**).
- **Скролл-параллакс** — `useBoardScroll` слушает реальный скролл-контейнер `.board`
  (не `window`) и пишет `--sv`; клавиша слегка деформируется/сдвигается.
- **Звук клика** — `useClickSound`: случайный сэмпл из пула `key_sound_1..4` (пул на вариант,
  чтобы быстрые клики не резались). Общий хук — тот же звук на тегах picker-режима.

### Пилюля-строка (`useNeuroBar.ts`)

Клик по клавише разворачивает пилюлю (Figma node 312:31933/312:31934, ширина 833px):

- **Текст** → Enter поднимает фуллскрин-чат с «мышлением» и blur-стримингом захардкоженного
  ответа. Esc-цепочка: чат → Spotlight → сворачивание строки.
- **Picker-режим** (иконка курсора) — активирует выбор блока страницы: по ховеру подсвечивает
  элементы с `data-ai-block`, клик добавляет тег-чип в строку. Блоки размечены на `/current`
  прямо в JSX, на Главной — рантаймом через `querySelector` (контент там `dangerouslySetInnerHTML`).
- **Voice-режим** (микрофон) — `getUserMedia` → Web Audio `AnalyserNode` → живой эквалайзер
  (26 столбиков реагируют на звук). По остановке вставляет случайную «распознанную» фразу.
  Отказ в доступе — мягкий откат к обычному полю.

### Тёмная тема

Тумблер «Оформление → Тёмная тема» (скоуп `body.aik-dark`):

- **keycap перекрашивается в чёрный** на уровне вершинных цветов — вторая палитра `dark` в
  `keyCapMesh.ts`, свап атрибута `color` **без перезагрузки GLB** (`applyKeyTheme`). Обе
  палитры считаются один раз при загрузке.
- **Пилюля** — тёмное стекло, светлый текст/иконки; **хвост** → графит.
- **Фуллскрин-чат намеренно НЕ затемняется** (`.chat` вне скоупа `aik-dark`).

## Панель tools (`ToolsProvider` → `ToolsPanel`)

Живёт в root layout → переживает смену роута. Всё UI-состояние в одном контексте `ToolsCtx`:
`dashboard` (из pathname), `variant`, `v2State` (6 булей), `panelOpen`, `trailEnabled`,
`keyTheme`. Провайдер также синхронизирует состояние в URL (дип-линки).

Панель:
- **Селектор дашбордов** (карточки «ИП» / «Бухгалтер») — `router.push`, панель не закрывается.
- **Вид меню** v1/v2 (только на Главной) — меняет `variant` без смены роута.
- **Кнопка АИ** → тумблер хвоста при перетаскивании.
- **Параметры v2** (только v2 Главной) — 6 свитчей стека счетов → сайдбар через `useV2Page`.
- **Оформление** → тумблер тёмной темы (в самом конце).
- Открытие/закрытие FAB; **Esc** закрывает.
- **Рамка**: при открытии вьюпорт борда обрезается `clip-path` (1:1, не масштаб), вокруг чёрный фон.

## Как перенесена вёрстка (важно!)

Крупная статичная разметка Главной (сайдбары, контент, Spotlight) перенесена **байт-в-байт**
из оригинальных HTML через `dangerouslySetInnerHTML` — строки в
`components/dashboard/htmlPartials.ts` (автоген скриптом, нормализует пути ассетов в `/assets`).
Поведение навешивается типизированными хуками, опрашивающими DOM (`querySelector`) в `useEffect`
с полным cleanup. `/current` — уже обычный JSX.

> Правите статичную разметку — правьте оригиналы и перегенерируйте `htmlPartials.ts`, либо
> редактируйте строки напрямую. Данные для тегов picker-режима (`data-ai-block`) на Главной
> навешиваются рантаймом в `DashboardContent.tsx`, т.к. в статику их не вписать.

## Подводные камни

- **StrictMode (dev)** дважды прогоняет эффекты — во всех хуках строгий cleanup (снятие
  слушателей, `clearTimeout/Interval`, `cancelAnimationFrame`, `renderer.dispose()`).
- **`React.memo` на MainV1/MainV2 обязателен** — иначе апдейт контекста пере-инжектит
  статику через `dangerouslySetInnerHTML`, стирая JS-выставленные стили.
- **Не запускать `next build` при живом `next dev`** — общий `.next` ломается; лечится
  `rm -rf .next` + рестарт. По той же причине после HMR-правок эффектов/модулей возможны
  «застывшие» ошибки в консоли — валидны только после чистого рестарта dev.
- **Оверлеи** (`.nbar/.spot/.chat/.twk`, `.board`) — вне трансформируемого контента (fixed
  внутри `transform`-предка ломается); neuro-обёртка — `display:contents`.
- **3D-клавиша**: цвета запекаются в вершины (не материал/текстуры); смена темы = свап
  атрибута `color`, не reload GLB. Клик по канвасу, а не по мешу — иначе промах мимо
  геометрии молча не активирует строку.

## Деплой и автосинхронизация форка

- Прод — Vercel из `main`: пуш → авто `next build` → деплой. URL: https://nefor-dash-k.vercel.app
- **GitHub Action** `.github/workflows/sync-fork.yml` — каждые 6ч (и вручную) fast-forward'ит
  `main` от upstream `FatherofNations/nefor-dash` через `gh repo sync`. Каждый resulting-пуш
  Vercel деплоит автоматически. Fast-forward-only: свои коммиты не перезаписывает.
- Локальные правки поверх апстрима → перед пушем `git rebase origin/main` (апстрим-мёрджи и
  чужие коммиты прилетают в форк регулярно).
